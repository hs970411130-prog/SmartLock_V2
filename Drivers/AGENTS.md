# AGENTS.md — 驱动层 (Drivers/)

本目录存放所有外设驱动（drv_*.c/h）。驱动层只和 HAL 打交道，不依赖应用层和 FreeRTOS。

> 通用约束（分层依赖、ISR 安全、Flash 写入校验、data_ver 等）见根 `AGENTS.md`；外设术语见根 `CONTEXT.md`。

## 本层专属规则

1. **通信类操作必须返回错误。** SPI 收发、I2C 读写、UART 带超时应答——返回 int，0 成功，负数错误码。纯 GPIO 操作、显示绘制可以返回 void。
2. **每驱动私有错误码。** 在自身 drv_xxx.h 内定义错误码枚举，不强求全局统一。App 层只判断 0/非 0，不解析具体错误码。
3. **静态持句柄。** 外设句柄用 static 全局变量持有。
4. **避免忙等。** 轮询状态的循环必须带 timeout。驱动内只做短超时轮询（≤10ms），长等待交由 App 层用 vTaskDelay。
5. **新增驱动不影响现有。** 不修改 app_config.h、app_types.h 和 app_ipc.h 中已有的宏和结构体。
6. **驱动层不持有业务数据。** 只传硬件数据（字节数组、GPIO 电平），不管业务语义。
7. **硬件状态枚举对外暴露。** App 层需要区分故障类型时，驱动在自身头文件提供纯标准 C 枚举，App 只 include 该驱动头文件，不触碰 HAL。
